BBR简述

  • Google 开源了其 TCP BBR 拥塞控制算法，并提交到了 Linux 内核，从 4.9 开始，Linux 内核已经用上了该算法。
  • 目的是要尽量跑满带宽，并且尽量不要有排队的情况。
  • 可以起到单边加速TCP连接的效果。
  • TCP BBR致力于解决两个问题
    • 在有一定丢包率的网络链路上充分利用带宽
    • 降低网络链路上的 buffer 占用率，从而降低延迟
