摘要算法

简介

Python 的 hashlib 提供了常见的单向加密算法(即摘要算法)，如MD5，SHA1等等。

摘要算法又称哈希算法、散列算法。它通过一个函数，把任意长度的数据转换为一个长度固定的数据串（通常用16进制的字符串表示）

摘要算法的特征

  • 输入一样，输出必然一样
  • 雪崩效应：输入的一点改变，引起结果的巨大改变，常用于防暴力破解
  • 定长输出：无论输入的长度多长，输出的结果都一样长
  • 不可逆：无法根据特征码还原数据。
  • 无法进行身份验证和保证数据的机密性

例如，Jerry 给 Tom 编写了一封信，内容为 Hello Tom - by Jerry ，并使用大家都公用的摘要算法附上文件的摘要信息994f6db481a806761ded4091c910d3e9 ，如果这封信在转交途中被其他人修改为 I hate you Tom - by Jerry ，当 Tom 收到信后并不知道内容已经被篡改，但是当他使用摘要算法算出了文件摘要，比对了被修改的信的摘要之后，便能十分肯定地确定信的内容被篡改了

经过 Salt 处理的 MD5 密码，只要 Salt 不被黑客知道，即使用户输入简单密码也很难通过 MD5 反推明文密码

但是如果有两个用户都使用了相同的简单密码比如 123456，在数据库中将存储两条相同的MD5值，这说明这两个用户的口令是一样的。有没有办法让使用相同密码的用户存储不同的MD5呢？如果假定用户无法修改登录名，就可以通过把登录名作为 Salt 的一部分来计算MD5，从而实现相同口令的用户也存储不同的 MD5

hmac

如果 Salt 是我们自己随机生成的，通常我们计算MD5时采用 md5(message + salt)。但实际上，把 Salt 看做一个 “口令”，加 Salt 的哈希就是：计算一段 message 的哈希时，根据不通口令计算出不同的哈希。要验证哈希值，必须同时提供正确的口令。

这实际上就是 Hmac 算法：Keyed-Hashing for Message Authentication。它通过一个标准算法，在计算哈希的过程中，把 key 混入计算过程中

和我们自定义的加 Salt 算法不同，Hmac 算法针对所有哈希算法都通用，无论是 MD5 还是 SHA-1。采用 Hmac 替代我们自己的 Salt 算法，可以使程序算法更标准化，也更安全。

使用hmac和普通hash算法非常类似。hmac输出的长度和原始哈希算法的长度一致。需要注意传入的 key 和 message 都是 bytes 类型，str类型需要首先编码为 bytes。
